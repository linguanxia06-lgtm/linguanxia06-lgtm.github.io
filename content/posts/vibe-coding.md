#

**首先，梳理想法**

-愿望清单：我想用AI编程做一个什么东西？是一个小游戏，博客，还是一个工作空间？有了需求之后，列出「愿望清单」，想实现什么功能。比如我想做个人网站，我希望它：首页展示个人简介，技能页展示我各项技能熟练程度，有一个作品集界面展示写的文章，做的项目。还要放上联系方式，放各大社媒平台链接。希望网页简洁美观、配色高级、动画流畅。

-设计草图：用Figma、Excalidraw等工具画一个简易的「草图」，去coolors找点配色灵感。如果不想设计原型图，也可以直接找一些现有的网站作为设计的对标。

**其次，让大模型生成文档**

-把用「愿望清单」、「草图」发给Gemini，让它从产品经理的专业角度生成一份PRD产品文档。过程中要叫Gemini实时反馈，有不明确的地方，主动确认。

-叫 Gemini生成UI设计文档，用Markdown的格式画一个图，看看是不是自己想要的界面布局。

-根据上面的对话生成一份技术文档，选择成熟的技术架构。


**最后，跟Cursor对齐需求后执行开发**

-在Cursor建立文档库，把PRD、UI、技术文档以.md的文件格式丢给它。先用Agent模式，将 PRDs 文件夹 add 到 Cursor Chat 对话框，跟Cursor 对齐一下需求。
-切换到Plan模式去执行整个开发计划。


这样做的好处是可以避免Cursor过度工程化，减少bug，节省token，vibe  coding出来的东西更符合需求。


**总结一下，就是七大步骤：**


1. 梳理想法：列愿望清单 + 画简易草图

2. 生成PRD：用Gemini输出标准产品文档

3. 生成UI文档：用Gemini输出UI设计规范及页面布局

4. 生成技术文档：用Gemini确定技术栈，要求使用成熟框架

5. 与Cursor对齐：建立文件夹，把那3个文档以.md的格式复制过来

6. 让Cursor理解项目：Agent模式@PRDS 投喂文档，确认其理解无误

7. 制定计划并实施：使用Plan模式生成步骤，自动化构建
